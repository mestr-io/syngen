# Agents (Synthetic User Profiles)

This file defines the **roles and characteristics** of the synthetic users (Agents) that will populate the generated Slack export. The `syngen` program uses these definitions to create consistent, varied user profiles with realistic metadata.

## 1. Agent Attributes
Each generated user ("Agent") will possess the following attributes, derived from a combination of random seeds and structured templates:

-   **ID**: Slack-compatible User ID (e.g., `U02M3TMTV9B`).
-   **Name**: Unique username (e.g., `john.doe`).
-   **Real Name**: Full name (e.g., "John Doe").
-   **Email**: Professional or personal email (e.g., `john.doe@example.com`).
-   **Timezone**: Realistic timezone assignment (e.g., `America/Los_Angeles`, `Europe/London`).
-   **Avatar**: URL to a placeholder avatar service (e.g., Gravatar or generated identicon).
-   **Status**: Admin, Owner, Bot, or Regular User.

## 2. Activity Profiles (Behavior Simulation)
To create a realistic distribution of messages (Gaussian/Normal distribution), agents are assigned an **Activity Profile**. This determines their likelihood of posting messages in specific channels and at specific times.

### 2.1. The "Power User" (Top 10%)
-   **Description**: Highly active, posts frequently across multiple channels.
-   **Volume**: High message count.
-   **Distribution**: Broad channel coverage.
-   **Role**: Often admins or team leads.

### 2.2. The "Regular Contributor" (Middle 60%)
-   **Description**: Posts consistently but primarily in specific team channels (e.g., `#dev`, `#general`).
-   **Volume**: Moderate message count.
-   **Distribution**: Focused on 2-3 primary channels.

### 2.3. The "Lurker" (Bottom 30%)
-   **Description**: Rarely posts, mostly reads.
-   **Volume**: Low message count (0-5 messages).
-   **Distribution**: Only in `#general` or `#random`.

## 3. Data Sources (Faker Integration)
The `syngen` tool utilizes embedded data subsets from the Python `faker` library to generate these agents:

-   **Names**: Sourced from `faker.providers.person.en_US` (First Names + Last Names).
-   **Usernames**: Generated by combining first/last names (e.g., `f.lastname`, `firstname.l`).
-   **Emails**: Generated using `faker.providers.internet` patterns.
-   **Avatars**: Generated using deterministic hashes based on the user ID.

## 4. Bot Agents
In addition to human users, `syngen` can generate Bot users (e.g., "GitHub", "Jenkins").
-   **Attributes**: `is_bot: true`, specific app IDs.
-   **Behavior**: High volume of automated messages in specific channels (e.g., `#builds`).
